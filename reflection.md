# Reflection

## What Worked
Building this persona-based AI chatbot was an incredible exercise in applied prompt engineering. The most successful aspect was definitely structuring the system prompts with distinct sections: Persona Description, Few-shot Examples, Chain-of-Thought (CoT), Output Instructions, and Constraints. Giving each persona a unique background (e.g., Anshuman's focus on first principles and core CS, Abhimanyu's practical startup experience, Kshitij's approachable analogies) drastically shifted the model's tone. The few-shot examples proved particularly powerful; by providing three distinct Q&A pairs for each persona, the model learned exactly how to format its output and what tone to strike without needing endless descriptive paragraphs.

## The GIGO Principle in Action
The "Garbage In, Garbage Out" (GIGO) principle was immediately evident during early testing. When I initially tried a basic prompt like "You are Anshuman Singh, answer coding questions," the model gave generic, robotic responses that sounded like standard ChatGPT. It had no edge, no specific domain knowledge, and didn't push the user to think deeper. Once I applied rich, well-researched inputs—specifically instructing it to never give the code solution immediately and to focus on the "why"—the output transformed. The CoT instruction was also a game-changer. By forcing the model to "reason step-by-step internally before answering," the logic became much clearer and the analogies became more relevant. High-quality, specific input directly resulted in high-quality, persona-accurate output.

## Areas for Improvement
If I had more time, I would improve the system in a few ways. First, I would implement dynamic few-shot prompting, where the examples injected into the prompt are dynamically chosen based on the user's question (e.g., if the user asks about system design, show system design examples). Second, I would improve the memory management of the conversation. Currently, the entire history is sent, but for longer conversations, implementing a summarization technique for past turns would keep the context window focused on the immediate problem and reduce token usage. Finally, I would refine the personas further by feeding the model actual transcripts of their lectures or talks to capture their exact speech patterns and vocabulary even more accurately.
